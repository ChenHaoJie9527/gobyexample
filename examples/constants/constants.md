# constants 进阶学习笔记

这个示例表面上是在讲 `const`，但真正值得学习的核心是：

1. Go 的常量不仅仅是“不能改的值”
2. Go 存在“无类型常量”这一机制
3. 常量会在具体上下文中再决定自己的类型
4. 编译期常量计算和运行时变量计算不是一回事

如果你把这几个点学明白，后面看数值、类型转换、标准库函数调用时会轻松很多。

## 先看示例想表达什么

这段代码的教学重点主要集中在下面几行：

```go
const s string = "constant"
const n = 500000000
const d = 3e20 / n

fmt.Println(d)
fmt.Println(int64(d))
fmt.Println(math.Sin(n))
fmt.Println(math.Sin(b))
```

它分别演示了：

1. 如何声明常量
2. 常量表达式如何在编译期计算
3. 无类型常量如何在需要时自动适配类型
4. 为什么 `math.Sin(n)` 能成立

## 一、`const` 到底是什么

很多初学者会把 `const` 理解成“只读变量”，但这并不准确。

在 Go 中，`const` 更准确的含义是：

- 它必须是编译期就能确定的值
- 它不是运行时才产生的结果
- 它通常只能用于布尔、字符串、字符、数值这类可在编译期确定的字面量或表达式

例如：

```go
const s string = "constant"
```

这一句说明两件事：

1. 常量可以显式写类型
2. 字符串字面量是合法的常量值

但下面这种思路就不成立：

```go
// 这不是合法常量的典型来源
// const name = getUser()
```

因为函数返回值发生在运行时，不能当作常量。

## 二、这个示例里用到了什么技术

### 1. 常量声明

示例里同时出现了两种写法：

```go
const s string = "constant"
const n = 500000000
```

第一种是“显式类型常量”，第二种是“未显式指定类型的常量”。

这两者的差异非常重要，因为第二种通常会进入 Go 的“无类型常量”机制。

### 2. 常量表达式

```go
const d = 3e20 / n
```

这里不是普通变量在运行时做除法，而是编译器在编译期就把这个表达式求值了。

这意味着：

- 这类计算不依赖程序运行时状态
- 它属于常量系统的一部分
- 理解它时不要直接套用“普通 `float64` 变量运算”的直觉

### 3. 无类型常量

`n` 和 `d` 最值得学的地方是：它们一开始并没有立刻固定为某个具体数值类型。

也就是说，下面这句：

```go
const n = 500000000
```

并不等价于：

```go
var n int = 500000000
```

前者是无类型常量，后者是已经确定为 `int` 的变量。这个差异，直接决定了后面函数调用能不能通过编译。

### 4. 显式类型转换

```go
fmt.Println(int64(d))
```

这一句说明：当你需要把常量落到某个具体类型时，可以显式转换。

这里的重点不是“会写转换语法”，而是理解：

- 在转换前，`d` 仍然可能是无类型常量
- 在转换后，它就变成了明确的 `int64`

### 5. 上下文驱动的类型决定

```go
fmt.Println(math.Sin(n))
fmt.Println(math.Sin(b))
```

`math.Sin` 的参数类型是 `float64`。之所以这里可以直接传 `n`、`b`，不是因为 Go 会自动把所有整数都转成 `float64`，而是因为：

- `n`、`b` 是无类型常量
- 在函数调用这个上下文里，编译器看到 `math.Sin` 需要 `float64`
- 所以它可以把这个常量解释成 `float64`

这就是这个示例最核心的语言机制。

## 三、为什么 `math.Sin(n)` 能过，而 `var n` 不行

这一点非常容易混淆，建议单独记住。

下面这种写法可以：

```go
const n = 500000000
fmt.Println(math.Sin(n))
```

原因是 `n` 是无类型常量，它可以在这里被解释为 `float64`。

但如果你写成：

```go
var n = 500000000
fmt.Println(math.Sin(n))
```

通常就不行，因为：

1. `var n = 500000000` 会先把 `n` 推导成一个具体类型，通常是 `int`
2. Go 不会把一个普通 `int` 变量隐式转换成 `float64`
3. 所以调用 `math.Sin(n)` 会报类型不匹配

也就是说：

- Go 不擅长做“变量的隐式数值转换”
- 但 Go 允许“无类型常量在上下文中确定类型”

这两个结论看起来接近，实际上完全不是一回事。

## 四、值得重点学习的地方

### 1. 把“常量”和“只读变量”区分开

这是理解 Go 常量系统的起点。

`const` 不是给变量加锁，而是要求值必须在编译期可确定。

### 2. 学会识别无类型常量

看到下面这种代码时：

```go
const n = 500000000
```

要有意识地想到：这里不一定已经是 `int` 了。

### 3. 理解“类型由上下文决定”

这是 Go 在数字常量处理上很有特色的地方。同一个常量，在不同场景里可能会落到不同具体类型。

### 4. 理解编译期计算的意义

`const d = 3e20 / n` 提醒你：Go 的一部分数值运算其实发生在编译期，而不是程序运行时。

### 5. 理解显式转换的边界

`int64(d)` 虽然能转，但前提是结果必须在 `int64` 可表示范围内。否则会直接编译失败，而不是运行时报错。

## 五、常见的坑

### 1. 误以为 `const n = 500000000` 的类型就是 `int`

这通常是最常见的误解。它更准确地说是“无类型整数常量”，不是普通 `int` 变量。

### 2. 误以为 Go 会自动做数值类型转换

`math.Sin(n)` 可以，不代表下面也可以：

```go
var n = 500000000
math.Sin(n)
```

后者一般需要你自己写成：

```go
math.Sin(float64(n))
```

### 3. 把常量表达式和运行时浮点运算混为一谈

`const d = 3e20 / n` 是编译期常量表达式，它和“定义两个 `float64` 变量后再相除”的语义背景不完全一样。

### 4. 忽略类型转换的范围检查

常量转具体类型时，编译器会检查值能不能装进去。超范围会直接报编译错误。

### 5. 误以为任何东西都能写成 `const`

函数返回值、切片、map、结构体实例，通常都不能作为常量。`getUser()` 返回的是运行时结果，所以不能定义成常量。

### 6. 被示例中的非核心代码分散注意力

当前文件里还有：

```go
c := getUser()
fmt.Println(c)
```

这只是普通函数调用，不是常量系统的核心教学内容。学习时应优先关注 `s`、`n`、`d`、`b` 和 `math.Sin(...)`。

### 7. 直接照抄脚本命令可能失败

当前目录里的脚本写的是：

```sh
go run constant.go
```

但实际文件名是 `constants.go`。如果直接执行脚本里的命令，可能会因为文件名不一致而失败。

## 六、这个示例真正应该记住的一句话

这个例子最重要的不是“Go 有 `const`”，而是：

**Go 的无类型常量可以在具体上下文中再决定类型，因此它们比普通变量更灵活。**

如果你记住了这句话，再回头看：

- 为什么 `int64(d)` 可以
- 为什么 `math.Sin(n)` 可以
- 为什么 `var n` 不行

这些现象就会自然串起来。

## 七、可以自己动手验证的练习

你可以把这几个实验写进 `constants.go` 里自己观察编译结果：

1. 把 `const n = 500000000` 改成 `var n = 500000000`，再调用 `math.Sin(n)`
2. 把 `const d = 3e20 / n` 改成变量版本，观察类型和结果变化
3. 尝试把一个超大常量转换成 `int64`，观察编译器报错
4. 试着写 `const name = getUser()`，看编译器如何拒绝

这些实验能帮你把“语法记忆”变成“类型系统理解”。
